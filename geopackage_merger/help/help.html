Geopackage Merger

Geopackage Merger compares one main GeoPackage with one or more source GeoPackages before copying data across. It is designed to help identify duplicates, missing layers, field mismatches, invalid geometries and other issues before a merge is carried out.

Run checks only does not copy any data. Data is only copied when you click Validate and merge and the validation checks pass.

Step 1: Add the GeoPackages

Main GeoPackage

Use Browse... to select the main GeoPackage. This is the file that will receive the copied data.

Source GeoPackages

Use Add... to add one or more source GeoPackages. These contain the data you want to copy into the main GeoPackage.

  • Add... adds source GeoPackages to the list.
  • Remove removes the selected source GeoPackage from the list.
  • Clear removes all source GeoPackages from the list.

Step 2: Use the main buttons

  • Run checks only checks the selected files and produces a report. Nothing is copied.
  • Validate and merge runs the checks and, if no blocking issues are found, copies the data into the main GeoPackage.
  • Save report... saves the report as a text file.
  • Clear report clears the report panel without changing the selected GeoPackages.

Progress and status messages

When checks or merging are running, a progress bar appears underneath the main action buttons and above the report section.

A short status message is also shown in the same area:

  • Orange text means something needs checking, or the selected GeoPackages have changed and checks should be run again.
  • Green text means the checks passed or the merge completed successfully.
  • Red text means the merge or validation failed and the report should be reviewed.

Step 3: Read the report

After running the checks, review each section of the report.

Blocking issues – Problems that must be resolved before the merge can continue. These include missing layers, field mismatches, CRS or geometry type mismatches, invalid geometries and duplicate key values.

Warnings – Items that should be reviewed before merging. They may not always prevent a merge, but they could indicate potential problems.

Information – General information about the GeoPackages, such as empty layers being ignored, stored layer styles being detected or layers that will be created if the relevant setting is enabled.

Merge complete – Displayed after a successful merge and summarises how many features were copied. If backups are enabled, the backup location is also reported.

What the checks look for

  • Missing or unreadable GeoPackage files.
  • Source files accidentally selected as the main file.
  • Missing layers or attribute tables in the main GeoPackage.
  • Field name and field type mismatches.
  • CRS or geometry type mismatches.
  • Invalid or empty geometries.
  • Duplicate unique or key fields.
  • Raster tile tables, which are reported but not copied by this version.
  • Stored QGIS layer style records.

Source tracking after merging

When features are copied into the main GeoPackage, the plugin adds a field called geopackage_source to merged vector layers where needed.

This field stores the file path of the source GeoPackage that each copied feature came from. This makes it easier to trace features back to their original GeoPackage after several files have been merged.

Settings

Click the settings button to change how the merge behaves.
  • Create a timestamped backup of the main GeoPackage before copying creates a backup before any data is merged. This is recommended.
  • Create missing layers or attribute tables in the main GeoPackage allows the plugin to create missing items in the main GeoPackage when they exist in a source GeoPackage.
  • Copy QGIS layer styles stored in the source GeoPackages where possible copies stored QGIS styles from the source files into the main GeoPackage where possible.
  • Ignore empty source layers and attribute tables allows empty source items to be skipped rather than treated as a problem.
  • Saved settings are remembered between QGIS sessions, so options do not need to be reselected every time the plugin or project is reopened.

Closing the dialog

Closing the main dialog using the window X or the Close button resets the main GeoPackage selection, source list, report text, status message and progress bar.

Saved settings are not reset when the dialog closes.

Help button

Recommended workflow

  1. Select the main GeoPackage.
  2. Add the source GeoPackages.
  3. Click Run checks only.
  4. Review the report, status message and any blocking issues.
  5. Run the checks again if changes were made.
  6. Click Validate and merge once the report is clear.
  7. Save the report if you need a record of the checks or merge.
