Geopackage Merger

Geopackage Merger checks one main GeoPackage against one or more source GeoPackages, then copies compatible data into the main file. It helps identify duplicates, incompatible structures and damaged geometries before anything is merged.

Nothing is copied when you use Run checks only. Data is copied only after you select Validate and merge and no critical issues are found.

Before you start

  • Close other programs that may be editing the selected GeoPackages.
  • Use the most complete and trusted file as the Main GeoPackage.
  • The main file cannot also be selected as a source.
  • Keep the backup setting enabled unless you already have a secure copy.

Using the plugin

1. Select main GeoPackage

Select Browse... and choose the GeoPackage that will receive the data. The report refers to this file as Main.

2. Select source GeoPackages

Select Add... and choose one or more files containing data to copy. Each source is numbered in the list as 1, 2, 3 and so on.

If a source is removed, the remaining sources are renumbered automatically. Reports use the matching labels Source 1, Source 2 and so on.

3. Run the mandatory checks

Select Run checks only to create a report without changing any GeoPackage. Resolve critical issues, then run the checks again.

Mandatory checks

All checks run automatically. They do not need to be selected individually.

Files and contents

  • Checks that the main and source paths exist and use the .gpkg extension.
  • Prevents the same source being selected more than once.
  • Prevents the main GeoPackage being used as a source.
  • Detects missing vector layers and attribute-only tables.
  • Reports raster tile tables as warnings because raster layers are not copied.

Layer compatibility

  • Checks that matching layers use compatible coordinate reference systems.
  • Checks that matching layers use the same geometry type, such as point, line or polygon.
  • Checks field names, field types and required fields.

Geometry quality

  • Null, empty and invalid source geometries are critical.
  • The report identifies the source number, layer and feature FID.
  • Features with unusable geometry are never copied.

Duplicate identifiers

Configured identifiers such as context, trench, drawing, sample, intervention, section and small-find numbers are checked between the main and source files and between different sources. A matching configured value in different GeoPackages is critical.

Exact feature duplicates

An exact duplicate is critical when features are in the same named layer, have the same configured identifying value, have exactly matching geometry, and occur in different GeoPackages. Matches contained only within one GeoPackage are not reported by this cross-file check.

Fields and data types

Fields missing from Main

If a source contains a field that is missing from the matching main layer or table, the field is added to Main. Existing main records receive NULL in the new field.

Fields missing from a source

Imported records receive NULL, or the main field default where applicable. If the field is required, does not allow null values and has no usable default, the report shows a critical issue.

Different data types

The main field type is authoritative. When a source uses a different type, the plugin checks the actual values before merging.

  • Safe conversions are reported as Information and performed during the merge.
  • Numeric text such as "12.5" can be converted to a real number.
  • A real value can become an integer only when it is a whole number.
  • Blank text can become NULL when the destination permits it.
  • Values that cannot be converted safely are critical and prevent the merge.
  • Binary values cannot be converted to ordinary text safely.

Understanding the report

The Files section lists each full path once. Other sections use the shorter labels Main, Source 1, Source 2 and so on.

SectionMeaning
SummaryShows the number of critical issues and warnings.
Critical issuesMust be fixed before merging. No data is copied while any critical issue remains.
WarningsDo not prevent merging, but should be reviewed.
InformationExplains planned actions, safe field conversions and other useful details.
Merge completeConfirms the number of copied features and shows the backup path when a backup was created.

What happens during a merge

  1. All validation checks run again using the current selections.
  2. If a critical issue is found, the merge is cancelled before copying begins.
  3. A timestamped backup is created when the backup setting is enabled.
  4. Missing permitted fields are added and compatible values are converted.
  5. Features and attribute-table records are copied into Main.
  6. QGIS layers are refreshed after the merge.

Source tracking

Merged vector layers contain a geopackage_source field. It stores the full source path for each copied feature so its origin can be traced later.

Layer styles

  • Existing layers keep the styles already stored in Main.
  • If a missing layer is created, its source style is copied automatically where available.
  • Source styles do not overwrite styles belonging to existing main layers.

Settings

Use the settings button in the dialog or Plugins > Geopackage Merger > Settings.
  • Create a timestamped backup – makes a copy of Main before data is written. This is enabled by default and recommended.
  • Create missing layers or attribute tables – permits missing items to be created from a source. When disabled, a missing item is critical.
  • Ignore empty source layers and attribute tables – skips empty source items. This is enabled by default.
  • Settings are remembered between QGIS sessions.

Progress and status

The progress bar and nearby message show what the plugin is doing during checks and merging. Large layers may take longer because each feature and geometry must be examined.

  • Green – checks passed or the merge completed.
  • Orange – selections changed, critical issues were found, or attention is required.
  • Red – validation or merging failed unexpectedly. Review the report.

Common problems

ProblemWhat to do
Validate and merge is unavailableSelect Main and at least one source, then run the checks. Changing a selection means checks must be run again.
A missing layer or table is criticalEnable creation of missing layers and tables in Settings, or add the missing item to Main manually.
A field conversion failedUse the reported field and FID to correct the incompatible value in the source, then rerun the checks.
A duplicate identifier was foundReview the reported Main and Source FIDs. Correct or remove the duplicate before merging.
An invalid geometry was foundRepair or remove the reported source feature in QGIS, save the source and rerun the checks.
A raster table was reportedThe warning does not block vector merging, but the raster content will not be copied.
